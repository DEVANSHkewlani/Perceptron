03 // Memory PORT:8090
Agent: A-08 // agent:memory
Manages all four storage layers. Logs episodes, calculates outcome rewards, maintains semantic causal graph after each feedback cycle.
Redis — Working Memory In-memory short-term cache. Stores recent CognitiveEvents with TTL eviction, current agent mutex locks (prevents concurrent conflicts), and active anomaly context. Sub-millisecond access.
TimescaleDB — Episodic Memory Time-series hypertable of completed episodes (anomaly→plan→execution→outcome). Partitioned by time. Queried for pattern history and injected as context into LLM prompts for similar past incidents.
Neo4j — Semantic Memory Knowledge graph of entity causal relationships (DEPENDS_ON, RESOLVES, MONITORS). Planners query topology here to calculate blast radius. Feedback loop mutates edge confidence weights after outcomes.
Qdrant — Procedural Memory Vector similarity database of action playbooks. Planners retrieve nearest-match remediation templates via cosine embedding search. Success-rate scores are tuned per-playbook after feedback.
↖ 08 Feedback writes back here
Mutates Neo4j weights · Tunes Qdrant playbooks · Logs to TimescaleDB
04 // World Model PORT:8092
Maintains a live belief state of the entire observed system. Tracks active anomaly list, entity health scores, blast radius calculations, and causal graph snapshots from Neo4j.
Active Agents A-01 Monitor A-09 Prediction Monitor updates beliefs on anomaly detection. Prediction reads timeseries here for T+5/T+15/T+60 forecasts.
↓
active anomalies + situation brief
05 // Reasoning Engine PORT:8093
Dual-path engine: fast heuristic rules for known anomaly signatures, deep LLM prompt (with episodic context + prediction) for novel ones. Produces root-cause hypothesis and a ranked action recommendation.
Active Agents Planner Pool ×3 A-09 Prediction Any available Planner picks up the anomaly. Prediction supplies T+5/T+15 forecasts as prompt context.
↓
reasoning chain
06 // Planning PORT:8094
Compiles a multi-step remediation DAG. Scores candidate actions via Qdrant playbook similarity. Routes high-risk plans through a human approval gate. Acquires Redis locks before committing.
Active Agents Planner Pool ×3 — domain-agnostic Any planner handles DB / Service / Queue anomalies. Queries Neo4j topology + Qdrant playbook templates.
↓
approved plan
07 // Execution PORT:8095
Dispatches each plan step to target infrastructure via SSH, container, or service APIs. Holds Redis mutex per entity to prevent concurrent conflicting actions. Emits execution traces back to Kafka.
Active Agents Executor Pool ×3 — domain-agnostic Pool-balanced — any executor runs any plan step for any domain (DB, service, queue).
↓
outcome trace
08 // Feedback Loop PORT:8096
Compares pre/post system state to calculate a reward signal. Reinforces successful playbooks in Qdrant, penalises failures. Mutates Neo4j edge weights. Writes the full episode to TimescaleDB.
Active Agent A-08 // agent:memory Writes back to Neo4j, Qdrant, TimescaleDB. Also updates World Model beliefs — closing the causal loop.
↩ LOOP CLOSES: Feedback → Memory + World Model
A-09 Prediction PARALLEL LANE
Runs continuously in parallel. Reads time-series metric snapshots from World Model every 60s and issues forward-looking forecasts without blocking the main pipeline.
Algorithm
Holt-Winters Exponential Smoothing + FBProphet baseline. Fits additive seasonal trend. Computes ±1.5σ confidence bands for upper/lower bounds.
Forecast Horizons
T+5m — high-confidence, near-term early warning
T+15m — medium range, threshold alerts
T+60m — trend direction, low confidence
Outputs To
→ World Model — updates situation brief if threshold is breached
→ Reasoning — injects T+5/T+15 as LLM prompt context per anomaly
