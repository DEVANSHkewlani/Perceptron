Method 1 — Config file (YAML manifest)

What it is

You write a single YAML file that declares every source the perception layer should monitor. On startup, the perception layer reads this file and instantiates one adapter process per source entry.

sources: - id: svc:auth-service type: log path: /var/log/auth-service/*.log format: json poll_interval_ms: 500 - id: svc:api-gateway type: metric prometheus_url: http://prometheus:9090 targets: ["api-gateway"] threshold_rules: - metric: http_error_rate gt: 0.05 severity: high - id: db:postgres-primary type: database dsn: postgresql://user:pass@pg:5432/prod exporters: [pg_stat_activity, pg_stat_replication] - id: queue:orders type: queue broker: kafka://kafka:9092 topic: orders thresholds: consumer_lag_gt: 1000

Method 2 — Service discovery (Kubernetes / Consul)

What it is

The perception layer queries a service registry at startup and on a regular re-scan interval (e.g. every 60 seconds). Every service it finds gets auto-registered as a source with sensible defaults.

For Kubernetes: it calls the K8s API server (GET /api/v1/namespaces/production/services) and receives a list of all running services with their labels, ports, and health endpoints. It auto-creates an api source for every service that exposes port 80 or 443, and a metric source for every service that has the label prometheus.io/scrape: "true".

from kubernetes import client, config as k8s_config def discover_k8s_sources(): k8s_config.load_incluster_config() v1 = client.CoreV1Api() services = v1.list_namespaced_service(namespace="production") sources = [] for svc in services.items: labels = svc.metadata.labels or {} if labels.get("cognitive.monitor") == "true": sources.append({ "id": f"svc:{svc.metadata.name}", "type": "api", "health_url": f"http://{svc.metadata.name}/health", "poll_interval_ms": 5000 }) if labels.get("prometheus.io/scrape") == "true": sources.append({ "id": f"svc:{svc.metadata.name}", "type": "metric", "prometheus_job": svc.metadata.name }) return sources

Method 3 — Auto-detection (active scanning)

What it is

The perception layer actively probes the environment to find sources that were not explicitly declared. Runs at startup and on a periodic schedule (every 5 minutes). Each probe type targets a different signal class.

class AutoDetector: def scan_prometheus_targets(self): # Queries Prometheus /api/v1/targets — returns every scrape target r = requests.get("http://prometheus:9090/api/v1/targets") for t in r.json()["data"]["activeTargets"]: self.register_if_new({ "id": f"svc:{t['labels']['job']}", "type": "metric", "prometheus_job": t["labels"]["job"] }) def scan_log_directories(self, watch_paths=["/var/log/services"]): # Uses inotify (Linux) to watch directories for new .log files import inotify.adapters inotify_obj = inotify.adapters.InotifyTree(watch_paths[0]) for event in inotify_obj.event_gen(yield_nones=False): _, types, path, filename = event if "IN_CREATE" in types and filename.endswith(".log"): self.register_if_new({ "id": f"log:{filename.replace('.log','')}", "type": "log", "path": f"{path}/{filename}" }) def scan_database_ports(self, host_range="10.0.0.0/24"): # Tries connecting to known DB ports (5432, 3306, 27017, 6379) for port, db_type in [(5432,"postgres"),(3306,"mysql"),(27017,"mongo")]: if self.port_open(host_range, port): self.register_if_new({ "id": f"db:{db_type}-{port}", "type": "database", "db_type": db_type })
